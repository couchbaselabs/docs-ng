DP3 release notes

This is the third developer preview edition of Couchbase Query Language released March 2014.

New features

  • KEYS clause
  • BETWEEN and NOT BETWEEN clause
  • JOIN clause (based on primary keys)
  • NEST clause (based on primary keys
  • A second form of Case expression
  • Array functions: ARRAY_CONCAT(), ARRAY_LENGTH(), ARRAY_APPEND(), ARRAY_PREPEND(), and ARRAY_REMOVE()
  • Array slicing
  • Date functions: functions with Unix MILLIS timestamps
  • Type functions: IS_, TO_, and TYPE_NAME()

Changed features

  • FROM ... OVER ... query pattern is changed to FROM ... UNNEST ...
  • ANY ... OVER ... query pattern is changed to ANY ... SATISFIES ...
  • ALL ... OVER ... query pattern is changed to EVERY ... SATISFIES ...

Fixes

  • Issue: MB-10461 - If a needles lies at the haystack's end boundary, a SUBSTR query returns NULL.
  • Issue: MB-10394 - Unable to run queries for cbq where the URL doesn't have a slash at the end.
  • Issue: MB-10295 - Performance issue with the DISTINCT operator.
  • Issue: MB-9872 - If tuq is started for a non-initialized node, the error " Pool default does not exist " displays for all queries.
  • Issue: MB-9810 - RHS of the UNNEST clause must begin with an already seen alias.

Known issues

This release is a Developer Preview and is not meant to be used in Production.

  • Data manipulation language (DML) syntax elements are not available with this release, however, this capability is planned for the future.

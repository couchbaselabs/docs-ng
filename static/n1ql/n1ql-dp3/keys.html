KEYS clause

The KEYS clause is an optional clause that follows the FROM clause. The KEYS clause is based on primary keys within a bucket. Only values having those primary keys are included as inputs to the query.

Syntax

SELECT result-expr FROM data-source join-type JOIN from-path [AS] alias KEYS expr

Compatibility

Compatible with Couchbase Server 2.2

Example

The following example show single and multiple keys specified.

       
SELECT * FROM customer KEYS "acme-uuid-1234-5678"

SELECT * FROM customer KEYS [ "acme-uuid-1234-5678", "roadster-uuid-4321-8765" ]

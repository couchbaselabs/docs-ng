LIMIT clause

The LIMIT clause imposes a specific number of objects returned in a result set by SELECT. This clause must have a non-negative integer as upper bound. The OFFSET clause optionally follows the LIMIT clause. If an offset is specified, that many number of objects are omitted from the result set before enforcing a specified LIMIT. This clause must be a non-negative integer.

Syntax

LIMIT expr
LIMIT expr OFFSET expr

Compatibility

Compatible with Couchbase Server 2.2

Description

An OFFSET clause specifies a number of objects to be skipped. If a LIMIT clause is also present, the OFFSET is applied prior to the LIMIT. The OFFSET value must be a non-negative integer.

A LIMIT clause imposes an upper bound on the number of objects returned by the SELECT statement. The LIMIT value must be a non-negative integer.

Examples

The following examples shows the LIMIT and OFFSET clauses with the N1QL tutorial.

Name and age list: limit by 2

SELECT fname, age
    FROM tutorial 
        ORDER BY age 
            LIMIT 2

Returns:

{
  "resultset": [
    {
      "age": 18,
      "fname": "Fred"
    },
    {
      "age": 20,
      "fname": "Harry"
    }
  ]
}

Name and age list: limit and offset by 2

SELECT fname, age
    FROM tutorial 
        ORDER BY age 
            LIMIT 2 OFFSET 2

Returns:

{
  "resultset": [
    {
      "age": 40,
      "fname": "Jane"
    },
    {
      "age": 46,
      "fname": "Dave"
    }
  ]
}

Golf products list: limit and offset by 10

SELECT * FROM product 
    UNNEST product.categories as cat
        WHERE lower(cat) in ["golf"] LIMIT 10 OFFSET 10
